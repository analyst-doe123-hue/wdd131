Site Purpose

BuildStrong Construction is a prototype company site demonstrating a professional presence for a civil and construction contractor. The site informs potential clients about services, showcases project case studies (residential, commercial, infrastructure), collects contact inquiries, and lists references for images/content.

Planned Pages & Features

  • Home — company intro, mission, services overview, testimonials.
  • Projects — gallery of projects with category filters (residential, commercial, infrastructure), project detail modal.
  • Contact — contact form (name, email, message), company details, embedded map, and form data saved to localStorage as a demo.
  • References — list of image/content sources and attributions.
  • Dynamic features — client-side filtering, testimonials carousel, responsive image optimization, lazy loading.

Reason for Choosing Subject

Construction is an industry that benefits from clear visual presentation of past work. This project demonstrates HTML/CSS/JS skills while producing a practical site that could represent a real small construction company.

Target Audience

Local homeowners, small business owners, municipal clients, and potential partners looking for civil and construction services.

Scenarios

  1. Prospective homeowner: "Can I see examples of completed residential projects similar to my renovation?" — Answered by Projects page with filter by category & project detail pages showing before/after photos.
  2. Procurement officer: "What services does BuildStrong offer and how do I contact you for a tender?" — Answered by Home (services & mission) and Contact (company details + form).
  3. Photographer/credit checker: "Where did the images come from and who needs to be credited?" — Answered by References page with clear attributions.

Testing & Validation

  1. HTML validation (W3C) — ensure no structural errors.
  2. WCAG contrast check — buttons & text against background (aim AA minimum).
  3. Responsive checks — test at 320px, 480px, 768px, 1024px, 1440px.
  4. Performance — compress/serve optimized images, use loading="lazy" for offscreen images.
  5. SEO basics — meta tags, meaningful headings, descriptive alt attributes for images.
  6. Form testing — client-side validation, store demo submit data to localStorage for persistence during development.
  7. Cross-browser — Chrome, Firefox, Edge, Safari checks on desktop & mobile.

Implementation Notes

Project filtering

Client-side filtering using data attributes and JS (e.g., data-category="residential"), optional URL hash for sharable filters.

LocalStorage demo

Save contact form drafts locally and display last 5 testimonial messages. Example: save under key buildstrong_contact_draft.

Images

Use optimized JPEG/WEBP, provide 1x/2x srcset, and lazy-load. Keep production images in /assets/images/projects/.

Contact Form localStorage snippet (implementation)

// on input change
const key = 'buildstrong_contact_draft';
const form = document.querySelector('#contactForm');
form.addEventListener('input', () => {
  const data = {name:form.name.value,email:form.email.value,message:form.message.value};
  localStorage.setItem(key, JSON.stringify(data));
});
// restore on load
const saved = JSON.parse(localStorage.getItem(key) || 'null');
if(saved){form.name.value=saved.name||'';form.email.value=saved.email||'';form.message.value=saved.message||''}
